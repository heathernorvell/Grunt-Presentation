Grunt is a JavaScript Task runner. Meaning it can run specific tasks automatically including JavaScript tests, linting (checking code for errors), etc. Grunt has tons a plugins that allow you to do almost anything you can imagine: including compiling your css preprocessor, or reloading your local development page when you save. This presentation uses Grunt to run a local server!

How to install Grunt

Installing grunt is pretty easy, but you do have to do it from the command line.

There is no reason to be scared. Promise.

Step 1!

The major prerequisite for grunt is having node.js installed. This is SUPER simple, you go to the website and download their packages. Now you just need to make sure that nodejs is in your $PATH. To find out just run npm from your terminal

Installing Grunt

In order to install grunt run:
npm install -g grunt-cli
This will give you the Command Line Interface for grunt and will put it into the system $PATH.

An important thing to note is that this doesn't install grunt itself, it installs the commands so that you can run grunt. Grunt will need to be installed with a gruntfile.

Gruntfile

Now we'll talk about the gruntfile and package.json

package.json

First let's talk about package.json

This file is used to store the metadata for the modules that node needs for your project. Here is an example:

{ "name": "my-project-name",
"version": "0.1.0",
"devDependencies": {
"grunt": "~0.4.5",
"grunt-contrib-jshint": "~0.10.0",
"grunt-contrib-nodeunit": "~0.4.1",
"grunt-contrib-uglify": "~0.5.0" } }

Installing Grunt using package.json

Now you'll need to actually install grunt. To do this run:

npm install grunt --save-dev

This will install grunt locally (in your current project) and will save any devDependencies that you may have specified.

Gruntfile

Gruntfiles have the following bits:

  • A Wrapper
  • Configuration
  • Loading grunt plugins and tasks
  • Custom Tasks

This can be written in javascript or in coffeescript.

Wrapper Function

This is required for grunt to run:

module.exports = function(grunt){
//grunt codez
};

Project and task configuration

First you want to specifiy that package.json file you just created in order to get the correct packages

grunt.initConfig({
pkg: grunt.file.readJSON('package.json'),
})

The initConfig function basically says this is the initial configuration
